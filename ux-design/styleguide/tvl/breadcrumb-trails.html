Breadcrumb trails

The user needs to be able to navigate back and have an understanding of where she is in relation to the rest of the site.

Use when:

  • The page displayed is within a hierarchy of pages and is not the topmost page.
  • The user cannot easily navigate through the hierarchy via other local navigation methods. For example, if the page is fairly deep in a hierarchy, the breadcrumb maybe the simplest way to provide navigation.
  • The page may be arrived at from an external source (e.g., a search engine results page) and the user will need a sense of context.

Labels:

  • Where possible, labels should match the title of their corresponding page.
  • Separate each label with a greater-than sign ( > ).
  • Include the header of the current page as the last label in the breadcrumb.
  • Do not use the label "Home" for the topmost page. Instead use the specific name for the topmost page (e.g. "Flights" or "Holidays" rather than "Home".)

Links:

  • Link all labels in the breadcrumb except the last one (which corresponds to the title of the current page.)
  • Never link the greater-than sign ( > ) or the spaces that separate the labels.
  • The link should be styled the same regardless of whether it has been visited or not.

Other considerations:

  • The breadcrumb sometimes corresponds to the user's recent browsing history, but is not equivalent to it.
  • Dynamically include the titles of user-generated content pages as appropriate.
  • A vertical listing is confusing because it looks like a menu.
  • The links should always be presented as secondary information to the main navigation. Use a smaller font and non-obtrusive colours.
  • The breadcrumb should not be relied on as the sole indication of the page name.
  • Future stages should not be included, as these may differ according to the path a user selects.

Applying these guidelines to Travelocity

Below are examples of breadcrumbs for all categories, including those which do not currently use breadcrumbs. These rules have been applied to all:

  • Links to the site homepage have been removed.
  • Links to the category homepages have been added.
  • ‘Extras’ is always ‘Optional extras’ to ensure cross-category consistency.
  • ‘Results’ is always ‘Search results’ to ensure cross-category consistency.
  • ‘Details’ is always ‘Passenger details’ to ensure cross-category consistency.
  • Future steps have been removed.

Any other changes to specific breadcrumbs are listed in each category.

Breadcrumbs are shown for both the search results/browse page and the final step in the booking process for each section.
