Link decoration & Mouseover

By default all links are underlined and on mouse-over the text decoration disappear.

Link decoration exception

Although there are a few exceptions like the Hotel Name in the Hotel Path. The hotel name in the hotel card is not underline by default only when the user mouses over it.

Link affordances

To reinforce the actions of the links we added several icons to add more context.

Overlay

When the user interacts with a link that triggers a modal window the following icon should be used.

Pop-Up Window

When the user interacts with a link that triggers a popup window the following icon should be used.

Toggle Expand/Collapse (Parent or Primary)

When there's too much information to display a toggle icon should be added to expand and collapse the content. The icon should always remain in the same place for a better UX.

Toggle Expand/Collapse (Child or Secondary)

When there are two similar toggle functions nested or near to each other the secondary or child toggle icon should be used. The icon and link should always remain in the same place for a better UX.

Promotions

For the Hotel and X-Sell Path the golden coins icon will be added to the left side of the link in the hotel card.

History: Go back

The back arrow should be used whenever the user is going to be redirected to a previous page in the path.

Action: Moving Forward

The forward arrow should be used whenever the user is going to be redirected to a next page in the path.

Scroll Up/Down

These up and down arrow should be used when the application scrolls the user browser window up or down to a specific section of a page.

Close window

When the user opens a modal box from a link (e.g. "change your search" or "view flight details") the close link should have a close icon too.

close

Question

The question icon should be used when something is not clear enough or to add a more meaningful description.
